# SepticFilter — ディープラーニングベースの変形悪口検出フィルター 🤬🤖

**🌐 Available Versions:** [🇺🇸 English](/README.md) | [🇰🇷 한국어 (Korean)](/README_KR.md)

> 🏆 **三星SDS マルチキャンパス ビッグデータ活用コンペティション — 最優秀賞（1位）**（マルチキャンパス、2022年）  
> K-デジタルトレーニング データサイエンス課程 チームプロジェクト

> 📌 **後続プロジェクト：** このプロジェクトで培った韓国語ドメイン特化 NLP の知見（字母トークン化戦略・アンサンブル設計・コミュニティクローリング）は、翌年の [AntiPhishingNLP](https://github.com/fairyofdata/AntiPhishingNLP)（KISA 最優秀賞）のスミッシング検出モデル設計に直接活かされました。

固定キーワードリスト方式の限界を超え、字母分解・特殊文字置換などによる**変形悪口（回避表現）**まで検出するディープラーニングフィルターシステムです。

---

## 🎯 問題定義

従来のキーワードブロックリストフィルターは、以下のような文字レベルの回避手法を検出できません：

| 原語 | 変形表現 |
|---|---|
| 씨발（罵倒語） | ㅅㅂ |
| 개새끼（罵倒語） | 개ㅅㄲ |
| 미친（罵倒語） | 미1친 |

形態素解析器もこれらを OOV（未登録トークン）として処理してしまい、有害なシグナルを失います。

**リサーチクエスチョン：**  
> *「形態素解析器が認識できない変形悪口を、どうやって検出するか？」*

---

## 🏗️ コア設計：デュアルトークン化戦略 + ソフト投票アンサンブル

重要な洞察：`ㅅㅂ` は形態素解析器には不透明ですが、字母単位に分解する（`['ㅅ', 'ㅣ', 'ᴕ', 'ㅂ', 'ㅏ', 'ㄹ']`）とシーケンス構造が保たれ、モデルが悪口パターンを学習できます。

### 第1ステージフィルター — FastText + LSTM（字母レベル）
- **FastText** によるサブワードエンベディングで希少語・誤字に対応
- プリフィルターとして機能し、ここで未検出の場合は第2ステージへ進む

### 第2ステージフィルター — ソフト投票アンサンブル
| モデル | 正確度 | 強み |
|---|---|---|
| 字母レベル LSTM | 93.22% | 長距離依存関係と字母シーケンス文脈の学習 |
| 字母レベル CNN | 93.70% | 悪口の中に文字を挟んだローカル N-gram パターン（例：`시1발`）の検出 |

> ※ 形態素レベル LSTM も構築しましたが、`시@발` を `시`・`@`・`발` に分割して正常語を悪口と誤判定するリスクがあるため、最終アンサンブルから除外しました。

### 🚨 エンジニアリング上の課題と解決策

| 課題 | 解決策 |
|---|---|
| **IP バン** | 最小60秒 × 複数乱数積の動的スリープ間隔 + `User-Agent` ヘッダースプーフィング |
| **韓国語エンコーディング** | CSV 保存時に `utf-8-sig` エンコーディングを使用して文字化けを回避 |
| **ノイズ・重複** | `emoji` ライブラリでエモジ除去、正規表現で空白・改行をクリーニング、マージ後に重複排除 |

## 🖥️ サービスアーキテクチャ（Django）

| レイヤー | 詳細 |
|---|---|
| **バックエンド** | Django MVT パターン（`urls.py`・`views.py` ルーティング） |
| **モデルサービング** | `.h5` LSTM/CNN 重み + `.pkl` トークナイザーを `views.py` メモリにロードしてリアルタイム推論 |
| **データベース** | ユーザー入力 + 判定結果（1=悪口、0=正常）を MySQL に自動保存し、将来の再学習データとして活用可能 |
| **フロントエンド** | 入力フォームとリアルタイム判定結果を表示する HTML/JS Web UI |
